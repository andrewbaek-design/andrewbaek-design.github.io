What Admin Clerk Does

The agent that works
where APIs end.

Pharmacy back-office runs on systems built before the integration era: payer portals, PA forms, outcome submission interfaces, wholesaler ordering. Admin Clerk operates against those systems the way a human would, with credentials, browser navigation, and structured outputs, so the work gets done without anyone clicking through it.

Prior Auth

Prior authorization submission

PA forms filled, supporting documentation attached, and status checks polled on a cadence. When a payer's portal updates, Admin Clerk pulls the new status into MedMe so the patient's chart and the pharmacist's queue stay current.

Eligibility

Payer portal automation

Where eligibility runs through a payer portal instead of an EDI feed, Admin Clerk handles the lookup. Coverage details, copay, plan type, all returned to MedMe in the same shape as a 270/271 transaction.

Outcomes

Reporting & PSP coordination

Outcome reporting into manufacturer PSP portals, structured data submission, and any other "this needs to be submitted weekly" workflow handled by an agent instead of by a coordinator on a spreadsheet.

Safe By Design

Every action logged.
Every escalation human-reviewed.

Admin Clerk's every step is captured in an immutable audit log. The agent escalates to a human the moment it encounters a situation outside its envelope, with the screenshot, the form state, and the suggested next action ready for review. No silent failures, no surprise submissions.
