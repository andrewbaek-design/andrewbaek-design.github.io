## Build system notes

- `build.py` reads `src/**/*.html`, processes `{{> partials/x }}` includes recursively, processes `{{ var }}` substitutions from frontmatter, writes output to the project root.
- **Directory routing:** every non-index page builds to `<slug>/index.html` so clean URLs like `/us/about` work without server rewrite rules. The default Python http.server resolves a directory to its `index.html`.
- Add a new page → drop it under `src/`. Run build. Done.
- Cache busting: bump the `?v=N` query on `<link href="/assets/brand.css?v=N">` in pages that consume CSS changes.

All design tokens live in `assets/brand.css`. Color, type, spacing, components.

- `--indigo` (#063E54), `--indigo-dark` (#042B3B), `--matcha` (#C3C430), `--seashell` (#FFF7F2), `--ink`

- Card pattern: `.h-card` (light) / `.h-card--feature` (dark feature card with matcha accent)
- Type pattern: `.h-display` (page hero), `.h-display--xl` (extra-large home hero), `.kicker` / `.kicker--matcha` (eyebrow)
- Button pattern: `.btn.btn-primary` / `.btn.btn-ghost` / `.pill.pill--solid`
- Mockup pattern: `.mockup-host` wraps animated mockups at 1192×656 with `.frame-inner` scaled via JS
